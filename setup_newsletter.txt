STEP 1 — Install the only new library needed
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  pip install schedule

All other libraries (smtplib, yfinance, feedparser, etc.) are already
installed with the Finance Tracker dashboard.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 2 — Create a Gmail App Password
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

You CANNOT use your regular Gmail password.
You need a special 16-character "App Password" that bypasses 2FA.

── 2a. Enable 2-Step Verification (required first) ──────────────────

  1. Go to: https://myaccount.google.com/security
  2. Click "2-Step Verification"
  3. Follow the prompts (phone number or Google Authenticator)
  4. Complete and confirm it is enabled

── 2b. Generate an App Password ─────────────────────────────────────

  1. Go to: https://myaccount.google.com/apppasswords
     (If you don't see this page, 2-Step Verification is not enabled yet)
  2. In "Select app" dropdown → choose "Mail"
  3. In "Select device" dropdown → choose "Windows Computer"
  4. Click "Generate"
  5. A 16-character password appears (e.g.  abcd efgh ijkl mnop)
  6. COPY IT NOW — Google will never show it again

  ⚠  Remove all spaces before using it: abcdefghijklmnop


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 3 — Set Environment Variables on Windows
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Replace the example values with your real email and app password.

── Option A: Command Prompt (permanent — survives restarts) ─────────

  setx GMAIL_ADDRESS      "your.email@gmail.com"
  setx GMAIL_APP_PASSWORD "abcdefghijklmnop"
  setx RECIPIENT_EMAIL    "where.to.send@gmail.com"

  ⚠  Close and reopen your terminal after running setx.
  ✅  Verify:  echo %GMAIL_ADDRESS%   (should print your email)

── Option B: PowerShell (permanent — survives restarts) ─────────────

  [System.Environment]::SetEnvironmentVariable("GMAIL_ADDRESS",      "your.email@gmail.com", "User")
  [System.Environment]::SetEnvironmentVariable("GMAIL_APP_PASSWORD", "abcdefghijklmnop",     "User")
  [System.Environment]::SetEnvironmentVariable("RECIPIENT_EMAIL",    "where.to.send@gmail.com", "User")

  ⚠  Restart PowerShell after running these.
  ✅  Verify:  $env:GMAIL_ADDRESS   (should print your email)

── Option C: PowerShell — current session only (lost on close) ──────

  $env:GMAIL_ADDRESS      = "your.email@gmail.com"
  $env:GMAIL_APP_PASSWORD = "abcdefghijklmnop"
  $env:RECIPIENT_EMAIL    = "where.to.send@gmail.com"

  (Useful for quick testing; use Option A or B for permanent setup.)

── Option D: Windows GUI ─────────────────────────────────────────────

  1. Press Win + S → search "Edit the system environment variables"
  2. Click "Environment Variables…" button
  3. Under "User variables for <your name>", click "New" three times:

     Variable name           Variable value
     ─────────────────────   ──────────────────────────────
     GMAIL_ADDRESS           your.email@gmail.com
     GMAIL_APP_PASSWORD      abcdefghijklmnop  (no spaces)
     RECIPIENT_EMAIL         where.to.send@gmail.com

  4. Click OK on all dialogs
  5. Restart your terminal

TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌  "Authentication failed"
    → You used your regular Gmail password instead of the App Password
    → App Password is 16 characters, generated at myaccount.google.com/apppasswords
    → Make sure to remove spaces (e.g. "abcd efgh" → "abcdefgh")

❌  "Missing environment variables"
    → Reopen your terminal after using setx
    → Verify with:  echo %GMAIL_ADDRESS%  (cmd)  or  $env:GMAIL_ADDRESS  (PS)

❌  "ModuleNotFoundError: No module named 'schedule'"
    → Run:  pip install schedule

❌  Email arrives but lands in Spam
    → Open the email, click "Report not spam" in Gmail
    → After a few days it will reliably land in inbox

❌  Values show "—" in the email
    → Market may be closed or Yahoo Finance data is temporarily unavailable
    → This is normal outside NSE / NYSE trading hours

❌  "Connection refused" or network errors
    → Make sure the machine has internet access
    → Try again; Yahoo Finance and ET Markets occasionally have brief outages
